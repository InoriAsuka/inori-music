# Phase v4.1.1 — EQ 均衡器 + Gapless 无缝播放（v4.1.0 遗留收尾）

**版本**: v4.1.1  
**状态**: planned  
**方向**: Flutter 客户端（EQ 插件接入 + 队列重构）+ Go 服务端单元测试补全

## 背景

v4.1.0 的 `full_player_screen.dart` 和 `audio_handler.dart` 已完成，但 plan 中标注的 4 项未实现：
1. **10 段 EQ** — `just_audio_equalizer` 插件接入、预设选择器、频段 Slider
2. **Gapless 无缝播放** — 队列从 `ProgressiveAudioSource` 逐曲创建改为 `ConcatenatingAudioSource`
3. **presigned URL 提前刷新** — gapless 模式下下一曲 URL 在过期前 120s 自动替换
4. **Go handler 单元测试** — lyrics/uploadLyrics/getLyrics/patchTrack 的 handler 层测试

v4.1.1 的目标是把这四件事补齐，正式关闭 v4.1.0 的所有遗留项。

## 目标

- Flutter `flutter analyze` 0 errors，`flutter test` 新增 EQ 状态机 ≥ 6 cases
- Go `go test ./internal/httpapi/...` 新增 lyrics + replayGain handler tests ≥ 9 cases
- 播放队列无缝衔接（`ConcatenatingAudioSource`），曲目切换无 0.5s 以上空白
- 10 段 EQ 预设 + 自定义频段，Settings 「音频」section 完整 EQ UI

### Flutter 客户端 — Gapless + URL 刷新

- [ ] **队列重构为 `ConcatenatingAudioSource`**
  - 读 `services/mobile/lib/src/player/audio_handler.dart`
  - 新增 `ConcatenatingAudioSource _concatSource`，初始化时创建
  - `playQueue()` 不再逐曲创建 `ProgressiveAudioSource`，而是：
    1. 清空 `_concatSource.children`
    2. 遍历 trackIds，逐个 `AsyncAudioSource(Uri.parse(url))` 加入
    3. 调 `_player.setAudioSource(_concatSource)`
  - `enqueue()` / `enqueueNext()` / `removeFromQueue()` 操作 `_concatSource`
  - 文件：`services/mobile/lib/src/player/audio_handler.dart`

- [ ] **presigned URL 提前刷新**
  - `audio_handler.dart` 中监听 `_player.currentIndexStream`
  - 当 `currentIndex` 变化时，检查下一曲的 presigned URL 是否将在 120s 内过期
  - 过期则重新请求 `GET /tracks/{id}/playback`，调 `_concatSource.replaceAt(index, newSource)`
  - 文件：`services/mobile/lib/src/player/audio_handler.dart`

- [ ] **`PlayerNotifier` 适配**
  - 读 `services/mobile/lib/src/player/player_notifier.dart`
  - `playTrack()` / `playQueue()` 方法改为通过 `audioHandler` 的 `ConcatenatingAudioSource` 操作队列
  - 保留 `_postHistory()` 和 `processingStateStream` 的 auto-advance 逻辑
  - 文件：`services/mobile/lib/src/player/player_notifier.dart`
