# Phase v3.1.0 — 封面图（Artwork）

**版本**: v3.1.0  
**状态**: planned  
**方向**: 服务端字段扩展 + Flutter 封面展示

## 背景

v3.0.x 的 `_ArtworkPlaceholder` 降级为占位图标，因为：
1. 服务端 Album/Artist 无封面 URL 字段
2. Flutter 端 `artworkUrlProvider` 尚未实现

## 目标

- 服务端：Album 可关联封面 media object；暴露 presigned 封面 URL 端点
- 客户端：TrackListTile / MiniPlayerBar / FullPlayerScreen 展示真实封面

## 任务清单

### 服务端（Go `services/api/`）

- [ ] `catalog.Album` 新增 `ArtworkMediaObjectID string`（omitempty）
- [ ] `albums` 表 migration `010_album_artwork`：`ALTER TABLE albums ADD COLUMN IF NOT EXISTS artwork_media_object_id TEXT`
- [ ] `catalogpg.Repository`：Save/Get/List/Scan 同步新增列
- [ ] `catalog.Service.UpdateAlbum`：支持 `UpdateAlbumRequest.ArtworkMediaObjectId *string` 修改
- [ ] 新增 `GET /api/v1/catalog/albums/{id}/artwork` 端点
  - requireViewerAuth
  - 查 album.ArtworkMediaObjectID → 查 media object → 调 `storage.Service.GeneratePresignedURL`
  - 成功返回 `{"url":"...","expiresIn":900}`；无封面返回 404
- [ ] OpenAPI contract 更新：`CatalogAlbum.artworkMediaObjectId`、artwork 路径、`AlbumArtworkResponse` schema；bump version
- [ ] 单元测试：`getAlbumArtwork` handler（成功/无封面/无 catalog 503）

### Flutter 客户端（`services/mobile/`）

- [ ] 新增 `artworkUrlProvider(albumId)` — `AutoDisposeFamilyAsyncNotifier<String?, String>`
  - 调用 `GET /api/v1/catalog/albums/{id}/artwork`
  - 成功返回 URL 字符串，404/error 返回 null
- [ ] 修改 `_ArtworkPlaceholder`（`full_player_screen.dart`）：接 `artworkUrlProvider`，CachedNetworkImage，error 降级
- [ ] 修改 `TrackListTile` artwork 区域：若有 albumId 则接 artworkUrlProvider，否则占位图
- [ ] 修改 `MiniPlayerBar` 封面区域同上
- [ ] `flutter analyze` 保持 0 errors

## 非目标

- Artist 封面（Artist 页面暂不支持，跟进 v3.2.x）
- 封面上传 UI（由 admin web 负责，Flutter 仅消费）

## 后续候选

- 多尺寸封面（缩略图 vs 全尺寸，通过 `?size=thumb` query param）
- Artist 级别 artworkMediaObjectId
