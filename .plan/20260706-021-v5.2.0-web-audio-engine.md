# Phase v5.2.0 — Web 音频引擎对齐 II（ReplayGain / gapless / 状态持久化）

**版本**: v5.2.0
**状态**: completed（代码/自动化验证完成；e2e 实跑与人工听感验收待集成环境执行）
**完成日期**: 2026-07-14
**方向**: Web 客户端（`services/web/` 音频引擎重构）

## 背景

Web 播放器（`hooks/useAudio.ts`）目前是单 `HTMLAudioElement` 直连：无增益控制、
无下一曲预载（曲间有明显加载间隙）、无淡入淡出、刷新页面丢失全部播放状态。
v4.5.0 服务端已自动分析 `replayGainDb`，v4.7.0 Flutter 端已实际应用——web 是唯一
不消费响度归一化的客户端。

依赖关系：建议在 v5.0.0（签名流 URL）之后执行——WebAudio 的
`MediaElementAudioSourceNode` 要求音频资源 CORS 干净（`crossOrigin="anonymous"` +
服务端/S3 正确的 `Access-Control-Allow-Origin`），签名 URL 方案落地后再统一处理
CORS 语义，避免返工。与 v5.1.0 无依赖，可并行。

## 目标

- Web 端 ReplayGain 开关实际生效，与 Flutter 端听感一致
- 队列连播曲间间隙从「可感知的加载停顿」降到近似无缝
- 刷新页面后队列、当前曲目、播放位置可恢复
- 现有播放主流程（e2e）零回归

## 任务清单

### WebAudio 增益管线（`services/web/hooks/useAudio.ts`、`lib/audio/`）

- [x] 新增 `lib/audio/audioGraph.ts`：`AudioContext` + `MediaElementAudioSourceNode` +
      `GainNode` 封装；audio 元素设置 `crossOrigin = "anonymous"`；
      AudioContext 需在首次用户手势后 `resume()`（浏览器 autoplay 政策）
- [x] ReplayGain 应用：track 变化时读 `replayGainDb`（playback descriptor / track 数据），
      `gainNode.gain.value = clamp(10 ** (db / 20), 0.1, 2.0)`；用户音量仍走
      `audio.volume`，与增益正交
- [x] 设置页新增 ReplayGain 开关（localStorage 持久化），默认关闭
- [x] CORS 验证与文档：`streamTrack` 响应经同源 nginx 无 CORS 问题；S3 presigned 场景
      在 `docs/operations/` 补一节 bucket CORS 配置要求（`GET` + `Origin` 放行）；
      CORS 不满足时优雅降级——跳过 WebAudio 管线直连播放（无增益但不失声）

### 双元素 gapless + 淡入淡出（`hooks/useAudio.ts`）

- [x] 重构为双 `HTMLAudioElement` 轮换：当前曲播放中即预取下一曲 playback descriptor
      并 `preload` 到备用元素；`ended` 时切换激活元素立即 `play()`，
      间隙由「秒级」降到「毫秒级」
- [x] 预载时机：当前曲进度 > 50% 或剩余 < 30s 触发；跳过预载失败静默回退现有逐曲加载
- [x] 可选淡入淡出（对齐 Flutter v4.7.0「切歌淡入淡出」语义）：切换前后经各自
      GainNode 做 N 秒线性 ramp（`gain.linearRampToValueAtTime`），设置项共享同一开关文案
- [x] 签名 URL 过期处理：预载 URL 带 15 分钟有效期，切换时若已过期（长曲目暂停场景）
      重新获取 descriptor

### 播放状态持久化（`store/player.ts`）

- [x] zustand `persist` 中间件（或手动 localStorage 同步）：队列 trackIds、currentIndex、
