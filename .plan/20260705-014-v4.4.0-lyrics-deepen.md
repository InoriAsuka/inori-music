# Phase v4.4.0 — 歌词深化（逐字高亮 / 双语翻译 / 后台管理）

**版本**: v4.4.0
**状态**: planned
**方向**: 全栈（Go 服务端 schema 扩展 + Flutter 逐字渲染 + Admin Web 管理 UI）

## 背景

v4.0.0 打通了歌词从零到一的链路（LRC/SRT 上传、存储、同步滚动），但仅支持整行级时间戳：
`lrc_parser.dart` 只识别 `^\[(\d{2}):(\d{2})[.:](\d{2,3})\](.*)` 行首时间戳，不解析行内
`<mm:ss.xx>` 逐字增强标签；`lyric_line.dart` 模型只有 `{timestamp, text}`，无翻译字段；
`full_player_screen.dart` 的 `_LyricsList` 对每行渲染单一 `Text`，无逐字渲染、无双语第二行。
服务端 `LyricsResponse` schema 只有 `format`/`content`/`mediaObjectId`，无翻译或来源标注字段。
Admin Web `catalog/page.tsx` 目前零歌词相关 UI，歌词只能通过 API 直接调用上传，运营侧无法可视化管理。
v4.4.0 在现有歌词链路基础上做深度补强，不改变底层存储模型（仍是 media object + 明文 content）。

## 目标

- 支持 Enhanced LRC 逐字级时间戳解析与卡拉 OK 式渐变高亮
- 支持双语歌词（原文 + 翻译）同屏渲染，可开关
- 歌词标注来源（embedded/manual/lrclib 等），便于后续追溯与去重
- Admin Web 提供歌词上传/预览/删除的可视化管理入口

## 任务清单

### 服务端（Go `services/api/`）

- [ ] **`tracks` 表新增翻译媒体对象列**
  - migration `014_track_lyrics_translation.sql`：`ALTER TABLE tracks ADD COLUMN IF NOT EXISTS lyrics_translation_media_object_id TEXT`
  - `catalog.Track` 新增 `LyricsTranslationMediaObjectID string`（omitempty）
  - `catalogpg.Repository` Save/Get/List/Scan 同步新增列
  - 文件：`services/api/migrations/014_track_lyrics_translation.sql`, `services/api/internal/catalog/track.go`, `services/api/internal/catalog/postgres/repository.go`

- [ ] **歌词来源标注字段**
  - `catalog.Track` 新增 `LyricsSource string`（`embedded`/`manual`/`lrclib`，omitempty，默认 `manual`）
  - `uploadTrackLyrics` handler 写入时设置 `LyricsSource = "manual"`（当前唯一上传入口）
  - 文件：`services/api/internal/catalog/track.go`, `services/api/internal/httpapi/handler.go`（`uploadTrackLyrics`）

- [ ] **`POST /api/v1/catalog/tracks/{id}/lyrics` 扩展支持翻译文件**
  - `multipart/form-data` 新增可选字段 `translation`（同 `file` 校验规则：UTF-8 文本，≤512KB）
  - 有 `translation` 字段时存储为独立 media object，写入 `track.LyricsTranslationMediaObjectID`
  - 响应体新增可选 `translationMediaObjectId`
  - 文件：`services/api/internal/httpapi/handler.go`（`uploadTrackLyrics`）

- [ ] **`GET /api/v1/catalog/tracks/{id}/lyrics` 返回翻译与来源**
  - `track.LyricsTranslationMediaObjectID` 非空时读取翻译文件内容，填充 `LyricsResponse.translation`
  - `LyricsResponse.source` 填充 `track.LyricsSource`
  - 文件：`services/api/internal/httpapi/handler.go`（`getTrackLyrics`）

- [ ] **`DELETE /api/v1/catalog/tracks/{id}/lyrics` 级联清理翻译**
  - 同时清除 `LyricsTranslationMediaObjectID` 并删除对应 media object（若存在）
  - 文件：`services/api/internal/httpapi/handler.go`（`deleteTrackLyrics`）

- [ ] **OpenAPI contract 更新**
  - `LyricsResponse` schema 新增 `translation string?`、`source string?`（enum: embedded/manual/lrclib）、`translationMediaObjectId string?`
  - `UploadLyricsRequest`（或对应 multipart 描述）新增 `translation` 字段说明
  - 版本升至 `4.4.0`
  - 文件：`packages/api-contract/openapi/storage-admin.v1.json`

- [ ] **单元测试**
  - `uploadTrackLyrics`：带 translation 字段成功 201 / translation 格式非法 400
  - `getTrackLyrics`：有翻译时返回 translation 字段 / 无翻译时字段为空
  - `deleteTrackLyrics`：级联删除翻译 media object
  - 文件：`services/api/internal/httpapi/handler_test.go`

### Flutter 客户端（`services/mobile/`）

- [ ] **Enhanced LRC 逐字解析**
  - `lrc_parser.dart` 新增行内 `<mm:ss.xx>` 标签切分逻辑：将一行文本按内联时间戳切分为 `List<LyricWord>{offset, text}` 片段
  - 无逐字标签的行保持现有整行解析，`words` 字段为空
  - 文件：`lib/src/lyrics/lrc_parser.dart`

- [ ] **数据模型扩展**
  - `lyric_line.dart`：`LyricLine` 新增 `List<LyricWord>? words`（逐字片段，可空）、`String? translation`（对应翻译行）
  - 新增 `class LyricWord { final Duration offset; final String text; }`
  - 文件：`lib/src/lyrics/lyric_line.dart`

- [ ] **`lyricsProvider` 合并翻译响应**
  - 解析 `LyricsResponse.translation` 后按行数对齐拆分（换行符分割），逐行赋值到对应 `LyricLine.translation`
  - 行数不匹配时忽略翻译（避免错位），仅记录 debug log
  - 文件：`lib/src/lyrics/lyrics_provider.dart`

- [ ] **`_LyricsList` 逐字渐变高亮 + 双语渲染**
  - 当前行若 `words` 非空：按 `position` 与每个 word 的 `offset` 比较，用 `TextSpan` 分段渲染，已唱过片段用 primary color，未唱片段用 onSurfaceVariant，渐变过渡（`AnimatedDefaultTextStyle` 或逐 word `Opacity`）
  - 当前行若 `words` 为空：回退现有整行高亮（v4.0.0 行为不变）
  - `translation` 非空且用户开启双语开关时，当前行下方追加次级样式（更小字号、onSurfaceVariant 色）翻译 `Text`
  - 文件：`lib/src/player/full_player_screen.dart`（`_LyricsList`/`_LyricsPage`）

- [ ] **双语歌词开关**
  - Settings 新增「双语歌词」`SwitchListTile`，状态持久化到 `SharedPreferences`（复用 v4.1.0 已引入的持久化模式）
  - 新增 `bilingualLyricsProvider`（读取/写入该开关状态）
  - 文件：`lib/src/settings/settings_screen.dart`（或对应 Settings 分区文件）

## 非目标

- 第三方歌词库自动抓取匹配（`lrclib` 来源标注仅为预留枚举值，本阶段不实现自动拉取逻辑）
- 歌词编辑器（逐行时间轴微调 UI）
- 翻译的机器翻译自动生成（本阶段仅支持人工上传翻译文件）
- SRT 格式的逐字高亮（SRT 无逐字标签能力，维持整行高亮）

## 后续候选

- 歌词来源自动匹配（集成 lrclib.net 等公开歌词库 API，按 title+artist+duration 匹配）
- 歌词时间轴可视化编辑器（admin web，拖拽调整时间戳）
- 多语言翻译（不止一种翻译语言，需要翻译语言标签）
