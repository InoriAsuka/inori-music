# Phase v3.4.0 — 离线播放 + 下载管理

**版本**: v3.4.0  
**状态**: planned  
**方向**: Flutter 客户端本地下载与离线播放

## 背景

用户在无网络或弱网环境下无法使用播放功能。引入本地下载和离线播放能力，使应用在飞行模式下仍可播放已下载曲目。

## 任务清单

### 依赖引入

- [ ] `package:sqflite ^2.4.x` — 本地 SQLite（客户端元数据）
- [ ] `package:path_provider ^2.1.x` — 获取 `getApplicationDocumentsDirectory()`
- [ ] `package:connectivity_plus ^6.x` — 网络状态检测
- [ ] `package:http ^1.2.x` 或复用 `dio` — 流式下载（进度回调）

### 本地数据库

- [ ] `OfflineTrack{trackId, title, artistName, albumTitle, albumId, localPath, sizeBytes, downloadedAt}` 模型
- [ ] `OfflineDb`（sqflite helper）：`create / insert / query / delete`
- [ ] migration v1：`CREATE TABLE IF NOT EXISTS offline_tracks (...)`

### DownloadManager

- [ ] `DownloadNotifier`（Riverpod Notifier）状态：`Map<String, DownloadStatus>`
  - `DownloadStatus` = `idle | downloading(progress) | done | error`
- [ ] `startDownload(trackId)`：
  1. `GET /api/v1/catalog/tracks/{id}/playback` → url
  2. `dio.download(url, localPath, onReceiveProgress)` 流式下载
  3. 下载完成 → 写入 OfflineDb
- [ ] `deleteDownload(trackId)`：删除文件 + OfflineDb 记录
- [ ] 启动时从 OfflineDb 恢复状态

### PlayerNotifier 集成

- [ ] `resolvePlaybackUrl(trackId)` 优先查 OfflineDb，有本地文件直接返回 `file://localPath`
- [ ] 网络不可用时跳过 presigned URL 请求，直接走本地路径

### 离线模式检测

- [ ] `connectivity_plus` 监听 `ConnectivityResult.none` → `offlineModeProvider`（bool）
- [ ] 目录列表页：离线模式下仅展示已下载曲目
- [ ] 网络恢复时自动上报离线期间累积的 play history（队列 + 批量 POST `/me/history/batch-...`，待 server 端有此端点；v3.4.x 先本地队列，v3.5.x 上报）

### UI

- [ ] `TrackListTile` 长按菜单增加「下载」/「删除下载」选项（根据 DownloadStatus 判断）
- [ ] 下载进度指示：`CircularProgressIndicator` 覆盖在 artwork 上
- [ ] Settings → 「Offline Library」section：已下载曲目列表 + 删除全部

## 非目标

- 自动同步整个 Library（手动触发下载）
- DRM / 加密存储（本地文件明文，适合私有部署场景）
- 上传至服务端（只读下载）

## 后续候选

- 离线播放历史批量上报（待 server 批量 history 端点）
- 下载队列优先级
