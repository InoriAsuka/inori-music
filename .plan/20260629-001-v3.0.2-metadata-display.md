# Phase v3.0.2 — 元数据显示质量修复

**版本**: v3.0.2  
**状态**: planned  
**方向**: Flutter 客户端 — 艺术家名/专辑名 UUID 问题修复

## 背景

v3.0.1 中 `PlayerNotifier._makeMediaItem` 和 `TrackListTile` 的 `artist` 字段展示的是 UUID（如 `3f2a1b...`）而非真实名称，因为 `CatalogTrack` 只携带 `artistId` / `albumId`，未做 lookup。

## 目标

`MiniPlayerBar`、`FullPlayerScreen`、`TrackListTile` 均展示可读的 artist name 和 album title。

## 任务清单

- [ ] 新增 `ArtistCacheNotifier`（`AutoDisposeFamilyAsyncNotifier<String, String>`）
  - 入参：artistId
  - 调用 `GET /api/v1/catalog/artists/{id}`，返回 `artist.name`
  - keepAlive 300s，避免频繁重新请求
- [ ] 新增 `AlbumCacheNotifier`（`AutoDisposeFamilyAsyncNotifier<String, String>`）
  - 入参：albumId
  - 调用 `GET /api/v1/catalog/albums/{id}`，返回 `album.title`
- [ ] 修改 `PlayerNotifier._makeMediaItem`
  - `artist` 字段：先查 `_trackCache` 内有无 artistName，无则在构建后异步回填
  - `album` 字段：同上
- [ ] 修改 `TrackListTile`
  - subtitle: `ref.watch(artistCacheProvider(track.artistId))` → `AsyncValue.when(data: (n) => n, ...)`
- [ ] 修改 `MiniPlayerBar`
  - artist 展示：从 `state.mediaItem.artist` 取（已在 _makeMediaItem 填充）
- [ ] 修改 `FullPlayerScreen`
  - artist / album 行同上
- [ ] 补充 Widget 测试：`TrackListTile` 展示 resolved artist name

## 非目标

- 不修改服务端 API（track 响应中已有 artistId，客户端自行 lookup）
- 不做预加载（按需 lookup，缓存复用）

## 后续候选

- 批量 lookup（`artistIds` 列表 → 单次请求），待 server 端有批量 GET artist 接口后引入
