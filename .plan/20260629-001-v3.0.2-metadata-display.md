# Phase v3.0.2 — 元数据显示质量修复

**版本**: v3.0.2  
**状态**: done  
**方向**: Flutter 客户端 — 艺术家名/专辑名 UUID 问题修复

## 背景

v3.0.1 中 `PlayerNotifier._makeMediaItem` 和 `TrackListTile` 的 `artist` 字段展示的是 UUID（如 `3f2a1b...`）而非真实名称，因为 `CatalogTrack` 只携带 `artistId` / `albumId`，未做 lookup。

## 目标

`MiniPlayerBar`、`FullPlayerScreen`、`TrackListTile` 均展示可读的 artist name 和 album title。

## 任务清单

- [x] 新增 `ArtistNameNotifier`（`AutoDisposeFamilyAsyncNotifier<String, String>`）
  - 入参：artistId
  - 调用 `GET /api/v1/catalog/artists/{id}`（经 `CatalogRepository.getArtist`），返回 `artist.name`
  - keepAlive 300s，避免频繁重新请求
  - 文件：`lib/src/catalog/catalog_cache_providers.dart`
- [x] 新增 `AlbumTitleNotifier`（`AutoDisposeFamilyAsyncNotifier<String, String>`）
  - 入参：albumId
  - 调用 `GET /api/v1/catalog/albums/{id}`（经 `CatalogRepository.getAlbum`），返回 `album.title`
  - 文件：`lib/src/catalog/catalog_cache_providers.dart`
- [x] 修改 `PlayerNotifier._makeMediaItem`
  - `artist` 字段：先查 `_artistNameCache`，无则 background backfill via `artistNameProvider`
  - `album` 字段：先查 `_albumTitleCache`，无则 background backfill via `albumTitleProvider`
  - 新增 `_artistNameCache` / `_albumTitleCache` 两个 `Map<String, String>` 字段
- [x] 修改 `TrackListTile`
  - subtitle: `ref.watch(artistNameProvider(track.artistId))` → `AsyncValue.when(data/loading/error)`
  - error 分支回退展示原始 artistId
- [x] `MiniPlayerBar` 已正确读取 `mediaItem.artist`（notifier 修复后自动生效）
- [x] `FullPlayerScreen` 已正确读取 `mediaItem.artist`（同上）
- [x] 补充 Widget 测试：`TrackListTile` 展示 resolved artist name
  - 文件：`test/track_list_tile_test.dart`
  - 注：pre-existing generated-model compile error（缺失 `.g.dart`）导致无法单独运行，非本次引入

## 非目标

- 不修改服务端 API（track 响应中已有 artistId，客户端自行 lookup）
- 不做预加载（按需 lookup，缓存复用）

## 后续候选

- 批量 lookup（`artistIds` 列表 → 单次请求），待 server 端有批量 GET artist 接口后引入
