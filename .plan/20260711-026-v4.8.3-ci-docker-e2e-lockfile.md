# Phase v4.8.3 — v4.8.2 后 CI/Docker/Admin E2E 收尾修复

**版本**: v4.8.3
**日期**: 2026-07-11
**类型**: patch / bugfix

## 背景

v4.8.2 后继续排查 v4 封版残留问题时，发现 Flutter 生成客户端提交范围、Docker web/admin deps stage、Admin Playwright smoke 登录选择器仍存在真实缺陷。

## 修复内容

1. **Flutter generated client 源码缺失**
   - `services/mobile/.gitignore` 不再忽略 OpenAPI generator 输出的 Dart/doc/test 源文件。
   - 补交用户播放列表、歌词、artwork 相关 generated source。
   - `.g.dart` / `.freezed.dart` 仍保持忽略，由 build_runner 生成。

2. **Docker `npm ci` EUSAGE**
   - web/admin `package-lock.json` 根包版本从 4.7.0 刷新到 4.8.3。
   - Dockerfile deps stage 改为精确复制 `package-lock.json`，避免 `package-lock.json*` 掩盖缺锁问题。
   - 保持 builder stage 复制 `packages/api-contract` 与 `packages/ui`，支撑 codegen 与 `@inori/ui` file dependency。

3. **Admin smoke 登录选择器**
   - `services/admin/e2e/smoke.spec.ts` 登录 helper 改用 `getByLabel("Username")`、`getByLabel("Password")` 和 submit button role。
   - 避免依赖 `input[type="text"]` 这种脆弱结构选择器。

## 验证

- Flutter generated client 修复：干净 build_runner 输出 248 个文件，`flutter analyze --no-fatal-infos` 通过，`flutter test --no-pub` 87/87 通过。
- Docker deps-stage 等价布局：复现 lockfile 根版本不一致导致的 `npm ci` EUSAGE，并在刷新 lockfile 后清除。
- `npm --prefix services/web ci` 通过。
- `npm --prefix services/admin ci` 通过。
- `npm --prefix services/admin run lint` 通过。
- `npm --prefix services/admin run type-check` 通过。
