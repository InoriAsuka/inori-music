# Phase v5.0.0 — 安全加固与对外可用基线

**版本**: v5.0.0
**状态**: planned
**方向**: 服务端为主（安全机制）+ 三端客户端适配 + 部署配置硬化

## 背景

v1–v4 的隐含假设是「个人自部署、可信网络」。**v5 主轴 = 产品化/对外开放**：让系统达到
可以给家人朋友或公网小规模使用的安全基线。2026-07-06 审查确认的遗留项：

1. **会话 token 进入 URL**：流媒体认证把完整会话 token 放 query string——
   `streamTrack`（`handler.go:2402`）接受 `?token=`，`services/web/hooks/useAudio.ts` 与
   `services/mobile/lib/src/player/player_notifier.dart:91` 都在拼接。token 有效期 24h、
   可操作用户全部数据，却会进入 nginx/代理访问日志、浏览器历史。
   （背景：`<audio>` 元素无法携带 Authorization header，query 参数是必要形态，
   问题在于放进去的**凭证的权限与寿命**。）
2. **Meilisearch 密钥不安全默认**：`docker-compose.prod.yml` 的
   `MEILI_MASTER_KEY=${MEILI_SEARCH_KEY:-default-master-key}` 有兜底默认值。
3. **登录无速率限制**：`POST /api/v1/auth/login` 可无限暴力尝试（bcrypt 提供单次成本，
   但无累计惩罚）。
4. **CORS 未配置时全放行**：`INORI_CORS_ORIGINS` 未设置即反射任意 Origin（dev 便利，
   prod 误配就是全开）。
5. **过期会话无清理**：`SessionRepository.DeleteExpiredSessions` 接口已存在
   （`auth/types.go:83`）但服务端没有定期调用方，过期会话行在 PG 中永久累积。

已核实无需重做的部分（现状良好）：会话 24h TTL + SHA-256 哈希存储 + 撤销机制、
bcrypt 口令哈希、登录时间侧信道防护、`SafeObjectPath` 路径穿越防御、
constant-time 静态 token 比较。

## 目标

- 流媒体 URL 中不再出现会话 token；泄露一条流 URL 的影响范围 = 单曲目、数分钟内
- 生产 compose 无不安全默认值，关键密钥缺失时拒绝启动
- 登录暴力尝试受限速与退避约束
- 过期会话自动清理
- 三端客户端无缝切换到新流媒体认证，播放体验无回归

## 任务清单

### 服务端 — 签名流媒体 URL（`services/api/`）

- [ ] 新增 `internal/streamsign` package：HMAC-SHA256 对 `trackId + exp` 签名，
      密钥来自 `INORI_STREAM_SIGNING_KEY`（未配置时启动日志警告并回退派生自
      `INORI_ADMIN_TOKEN`，两者皆缺时禁用 stream 端点）；
      产出/校验 `?exp=<unix>&sig=<hex>` 参数对，有效期默认 15 分钟（与 S3 presign 对齐）
- [ ] `getTrackPlaybackDescriptor`（handler.go）：`streamUrl` 直接返回**已签名**的完整 URL
      （含 exp/sig），客户端不再自行拼接任何认证参数
- [ ] `streamTrack`：校验签名 + 过期，替代现有 `?token=` 会话校验；
      保留 Authorization header 会话路径（API 调试用），移除 `?token=` 查询参数支持
- [ ] OpenAPI：`TrackPlaybackDescriptor.streamUrl` 描述更新（删除"Append ?token=" 指引），
      契约版本升至 `5.0.0`
- [ ] 单元测试：签名正确 / 过期拒绝 / 篡改 trackId 拒绝 / 密钥未配置行为

### 服务端 — 登录限速（`services/api/internal/httpapi/`）

- [ ] `login` handler 前置内存限速器：per-IP（`X-Forwarded-For` 首个 IP，信任边界见
      nginx 配置）+ per-username 双维度；连续失败 5 次后按指数退避拒绝（60s 起），
      成功登录清零；返回 429 `too_many_requests`
- [ ] 限速状态仅存内存（单实例部署，不引入 Redis）；结构留接口便于未来替换
- [ ] 单元测试：连续失败触发限速 / 成功后清零 / 不同用户名互不影响 / 退避递增

### 服务端 — 会话清理

- [ ] `cmd/server/main.go` 启动每小时 goroutine 调用
      `SessionRepository.DeleteExpiredSessions(ctx, now-7d)`（保留 7 天供审计），
      优雅关停时退出

### 部署配置硬化（`docker-compose.prod.yml`、`infra/nginx/`）

- [ ] `MEILI_MASTER_KEY=${MEILI_SEARCH_KEY:?MEILI_SEARCH_KEY required}`（移除默认值，
      与 POSTGRES_PASSWORD 同模式）
- [ ] 新增 `INORI_STREAM_SIGNING_KEY` 环境变量注入
- [ ] nginx 配置追加安全响应头：`Strict-Transport-Security`、`X-Content-Type-Options: nosniff`、
      `X-Frame-Options: DENY`、基线 `Content-Security-Policy`（web/admin 需实测放行项）
- [ ] `INORI_ENV=production` 时若 `INORI_CORS_ORIGINS` 未配置，启动打 ERROR 级警告
      （不直接拒绝启动，同源 nginx 部署本就不需要 CORS）

### 客户端适配（三端）

- [ ] Web `hooks/useAudio.ts`：删除 `u.searchParams.set("token", token)` 逻辑，
      直接使用服务端返回的已签名 `streamUrl`；播放中 URL 过期的重取逻辑
      （`error` 事件 → 重新 GET playback descriptor → 恢复 position）
- [ ] Flutter `player_notifier.dart` `resolvePlaybackUrl`：同上删除 token 拼接与
      `?token=` fallback 分支
- [ ] Admin 无流媒体路径，仅回归验证
- [ ] e2e：web 播放主流程在新认证下通过

### 验证

- [ ] Go 全量测试 + `-race` 绿
- [ ] 人工验证：nginx access log 中流媒体请求不再出现会话 token；
      拷贝一条流 URL 在 16 分钟后访问返回 401

## 非目标

- OAuth/OIDC、第三方登录（候选）
- refresh token / 会话滑动续期机制（24h TTL 对当前规模够用）
- 多租户/组织模型
- 审计日志域（候选）
- HTTPS 证书自动化（部署环境职责，文档说明即可）

## 后续候选

- 登录限速持久化（Redis/PG），支持多实例
- 管理端「安全概览」页：活跃会话全局视图、近期失败登录
- 媒体访问审计日志（谁在何时播放了什么）
- CSP 从基线收紧到严格模式（nonce 化内联脚本）
