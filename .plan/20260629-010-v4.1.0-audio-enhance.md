# Phase v4.1.0 — 音频增强（EQ / 响度归一化 / Gapless / 速度控制）

**版本**: v4.1.0  
**状态**: planned  
**方向**: 全栈（Go 服务端 ReplayGain 元数据 + Flutter 音频引擎扩展）

## 背景

v3.x 的音频播放为"基础播放"：无音效处理、逐曲创建 AudioSource（不支持无缝衔接）、
无播放速度控制。v4.1.0 在音频引擎层引入四项增强，全部持久化到 Settings，冷启动恢复。

依赖 v4.0.0 完成（播放器 UI 框架已稳定，Settings 页面已存在）。

## 目标

- 10 段参数均衡器（EQ），预设 + 自定义，实时生效
- ReplayGain 响度归一化，服务端存储元数据，客户端应用增益
- Gapless 无缝播放（队列切换无空白）
- 播放速度 0.5×–2× 控制

### Flutter 客户端（`services/mobile/`）

#### 依赖

#### EQ 均衡器

- [ ] **`EqSettings` 模型**
  - `lib/src/audio/eq_settings.dart`：`class EqSettings { final bool enabled; final List<double> bands; /* 10 bands, dB -12 to +12 */ final String preset; }`
  - 预设常量：`Flat`（全 0）、`BassBoost`（低频 +6dB）、`Vocal`（中频 +3dB）、`Electronic`（低+高 +4dB，中 -2dB）

- [ ] **`EqNotifier`**（`Notifier<EqSettings>`）
  - 初始化时从 `SharedPreferences` 恢复
  - `setEnabled / setBand / setPreset` 方法，每次变更持久化
  - 变更时调用 `AudioHandler` 更新 equalizer 参数
  - 文件：`lib/src/audio/eq_notifier.dart`

- [ ] **EQ UI（Settings 「音频」section）**
  - 开关（`Switch`）+ 预设下拉（`SegmentedButton` 4 预设）
  - 10 条垂直 `Slider`，频段标签（31Hz / 62Hz / 125Hz / 250Hz / 500Hz / 1kHz / 2kHz / 4kHz / 8kHz / 16kHz）
  - EQ 关闭时滑块 disabled（灰色）
  - 文件：`lib/src/settings/settings_screen.dart`（新增 section）

#### ReplayGain 响度归一化

- [ ] **`ReplayGainNotifier`**（`Notifier<bool>`）
  - 存储开关状态到 `SharedPreferences`（key `audio.replayGain`）
  - 文件：`lib/src/audio/replay_gain_notifier.dart`

- [ ] **AudioHandler 应用 ReplayGain**
  - 在 `playMediaItem()` 中，若 `replayGainNotifier.state == true` 且 `track.replayGainDb != null`，
    计算线性增益 `gain = pow(10, replayGainDb / 20)`，调 `_player.setVolume(gain.clamp(0.0, 1.0))`
  - 若 ReplayGain 关闭或字段为 null，`setVolume(1.0)`
  - 文件：`lib/src/audio/audio_handler.dart`

- [ ] **Settings UI**：ReplayGain 开关追加到「音频」section

#### Gapless 无缝播放

- [ ] **播放队列改用 `ConcatenatingAudioSource`**
  - `PlayerNotifier.playQueue()` / `enqueue()` / `enqueueNext()` 改为操作 `ConcatenatingAudioSource`
  - `AudioHandler.init()` 预创建 `ConcatenatingAudioSource`，并在 `setAudioSource` 时一次传入
  - 注意：presigned URL 有效期（900s），超时前需刷新；在 `currentIndexStream` 监听中，提前 120s 刷新下一曲 URL（`ConcatenatingAudioSource.removeAt + insert`）
  - 文件：`lib/src/audio/audio_handler.dart`、`lib/src/player/player_notifier.dart`

#### 播放速度控制

- [ ] **`SpeedNotifier`**（`Notifier<double>`，初始 1.0）

  - `setSpeed(double)` → 校验范围 `[0.5, 2.0]` → 持久化 → 调 `AudioPlayer.setSpeed()`
  - 文件：`lib/src/audio/speed_notifier.dart`

- [ ] **`FullPlayerScreen` 速度控制按钮**

  - 当前速度高亮（primary color）
  - 文件：`lib/src/player/full_player_screen.dart`

#### 持久化 & 冷启动恢复

- [ ] **`AudioSettingsRestorer`**（`ProviderObserver` 或 `app.dart` 初始化逻辑）

## 非目标

- 空间音效（立体声展宽、混响）
- 参数均衡器以外的动态处理（压缩、限幅）
- ReplayGain 分析工具（`replayGainDb` 值由外部工具分析后通过 admin API 写入，不在 App 内计算）
- Windows / Linux EQ（`just_audio_equalizer` 平台支持范围以库文档为准，不支持的平台静默禁用 EQ）
