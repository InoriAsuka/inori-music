# Phase v5.3.0 — Web 体验对齐 III（用户播放列表 + 速度控制 + 睡眠定时器）

**版本**: v5.3.0
**状态**: completed
**完成日期**: 2026-07-16
**方向**: Web 客户端（`services/web/`，消费既有服务端能力，无服务端变更）

## 背景

2026-07-06 审查确认三块**服务端能力已就绪但 web 端零消费**的功能缺口：

1. **用户播放列表**：v3.2.0 已交付 `/api/v1/me/playlists` 全套端点（CRUD + 曲目增删排序），
   Flutter 端已有完整 UI；`services/web/` 全目录 grep `me/playlists` **零命中**——
   web 用户无法创建/管理自己的播放列表（现有 `app/(app)/playlists/` 页面是服务端
   管理的 catalog playlist，只读浏览）。
2. **播放速度控制**：v4.1.0 Flutter 端交付（0.5×–2×）；web 端 grep `playbackRate` 零命中。
3. **睡眠定时器**：v4.2.0 Flutter 端交付（固定时长 + 曲目结束模式）；web 端零实现。

web 已依赖 `@dnd-kit/core`/`@dnd-kit/sortable`（`package.json`），播放列表拖拽排序
基建现成。本阶段与 v5.1.0/v5.2.0 无硬依赖，可并行；建议排在 v5.2.0 之后执行，
速度控制可直接落在双元素引擎上（两个 `HTMLAudioElement` 都要设置 `playbackRate`）。

## 目标

- Web 用户可创建/重命名/删除自己的播放列表，向其中增删曲目并拖拽排序，一键播放全部
- Web 播放器支持 0.5×–2× 速度切换，设置持久化
- Web 支持睡眠定时器，语义与 Flutter 端一致（固定时长 / 当前曲目结束后停止）

- [x] `library/playlists/page.tsx`：列表页——GET `/api/v1/me/playlists`，
      新建（命名对话框 → POST）、重命名（PATCH）、删除（确认 → DELETE）

- [x] `TrackRow` 上下文菜单（web 各曲目列表页共用）新增「添加到播放列表」：
      弹出选择器（列出我的播放列表 + 内联新建），POST `.../tracks`

- [x] 空状态与错误态（无播放列表引导创建 / 404 回退列表页）

### 播放速度控制（`services/web/components/player/PlayerBar.tsx`、`hooks/useAudio.ts`）

- [x] PlayerBar 新增速度菜单（0.5× / 0.75× / 1× / 1.25× / 1.5× / 2×，对齐 mobile 档位）
- [x] `useAudio` 应用 `audio.playbackRate`；v5.2.0 双元素引擎下当前与预载元素都设置，
      切换元素时继承当前速度
- [x] 速度设置持久化 localStorage，非 1× 时 PlayerBar 显示当前倍速角标

### 睡眠定时器（`services/web/store/` + PlayerBar）

- [x] `store/sleepTimer.ts`（zustand）：固定时长（15/30/45/60 分钟）与
      「当前曲目结束后停止」两种模式；倒计时状态；到期调用 player store `pause()`
      并自动清除——语义对齐 mobile `SleepTimerNotifier`
- [x] 「曲目结束」模式挂接 `useAudio` 的 `ended` 事件（先 pause 后不 skipToNext）
- [x] PlayerBar 定时器图标入口 + 激活时倒计时角标；页面刷新后定时器不恢复（明确行为）
