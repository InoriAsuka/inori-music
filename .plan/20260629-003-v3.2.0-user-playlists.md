# Phase v3.2.0 — 用户个人播放列表

**版本**: v3.2.0  
**状态**: planned  
**方向**: 服务端新域 + Flutter Library 功能扩展

## 背景

当前 catalog playlist 是 admin 管理的（Phase 46-49），viewer 只能读取。用户需要自建播放列表来管理个人收藏顺序。

## 目标

viewer 可创建/管理自己的播放列表，与 admin catalog playlist 完全独立。

## 任务清单

### 服务端（Go `services/api/`）

- [ ] 新增 `internal/userplaylist` 包
  - 类型：`UserPlaylist{ID, UserID, Name, Description, TrackIDs []string, CreatedAt, UpdatedAt}`
  - `Repository` 接口：Save / Get / ListByUser / Delete
  - `Service`：CreatePlaylist / ListPlaylists / GetPlaylist / UpdatePlaylist / DeletePlaylist / AddTrack / RemoveTrack / SetTracks
- [ ] `MemoryRepository` 实现（用于单元测试）
- [ ] `postgres.Repository` 实现
  - migration `011_user_playlists`：`user_playlists(id, user_id, name, description, created_at, updated_at)` + `user_playlist_tracks(playlist_id, position, track_id)`
- [ ] HTTP 路由（requireViewerAuth，user_id 从 auth context 注入）
  - `POST   /api/v1/me/playlists` → createUserPlaylist（name 必填）
  - `GET    /api/v1/me/playlists` → listUserPlaylists（limit/offset/sortBy）
  - `GET    /api/v1/me/playlists/{id}` → getUserPlaylist
  - `PATCH  /api/v1/me/playlists/{id}` → updateUserPlaylist
  - `DELETE /api/v1/me/playlists/{id}` → deleteUserPlaylist
  - `POST   /api/v1/me/playlists/{id}/tracks` → addTrackToUserPlaylist
  - `DELETE /api/v1/me/playlists/{id}/tracks/{trackId}` → removeTrackFromUserPlaylist
  - `GET    /api/v1/me/playlists/{id}/tracks` → getUserPlaylistTracks（分页）
  - `PUT    /api/v1/me/playlists/{id}/tracks` → setUserPlaylistTracks（整体替换）
- [ ] OpenAPI contract 更新：`UserPlaylist` schema + 9 个路径；bump version
- [ ] 单元测试：Service（8 cases）+ handler（10 cases）

### Flutter 客户端（`services/mobile/`）

- [ ] `UserPlaylistNotifier`（Riverpod AsyncNotifier）：list / create / delete / addTrack / removeTrack
- [ ] Library Tab 新增「我的播放列表」section
  - 列表 + 新建按钮（showDialog → 输入名称）
- [ ] `UserPlaylistDetailScreen`（路由 `/library/playlists/:id`）
  - TrackListTile 列表（分页）
  - 顶部「播放全部」按钮
  - 编辑/删除入口
- [ ] 长按 `TrackListTile` BottomSheet：「添加到播放列表」→ 选择或新建
- [ ] `flutter analyze` 保持 0 errors

## 非目标

- 播放列表封面图
- 播放列表排序拖拽（ReorderableListView 留 v3.2.x 迭代）
