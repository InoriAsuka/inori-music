## 任务清单

### Critical / High — 功能阻断或数据安全

- [ ] **fix(offline): 下载时跳过认证头注入**
  - 根因：`dioProvider` 的拦截器对每个请求无条件注入 `Authorization: Bearer`，
    导致 presigned S3/MinIO URL 同时携带 query-string 签名和 Header 签名，服务端返回
    `SignatureDoesNotMatch(403)`，离线下载功能对所有用户完全失效。
  - 修复：在 `startDownload` 内用不带拦截器的裸 Dio 实例（或在现有 Dio 上用
    `options.extra['skipAuth'] = true` + 拦截器放行逻辑）发起 presigned URL 的下载请求。
  - 文件：`services/mobile/lib/src/offline/download_notifier.dart`
  - 参考：`services/mobile/lib/src/api/api_client.dart:52-54`

- [ ] **fix(api/userplaylist): AddTrack/RemoveTrack/SetTracks 并发写竞争**
  - 根因：三个方法均为"读-改-写"两段式操作，Save() 事务不覆盖前序 GetPlaylist，
    并发请求的最后写者会静默覆盖先者的修改，导致 track 丢失。
  - 修复：在 Postgres 实现中，Save() 内改用 `SELECT … FOR UPDATE` 锁行后再操作；
    或在 service 层为每个写操作引入乐观锁（version 列 + 重试），Memory 实现加 mutex 覆盖读写全程。
  - 文件：
    - `services/api/internal/userplaylist/service.go`
    - `services/api/internal/userplaylist/postgres/repository.go`
    - `services/api/internal/userplaylist/memory_repository.go`

- [ ] **fix(api/userplaylist): Get() 将所有 Scan 错误映射为 ErrNotFound**
  - 根因：`row.Scan(…)` 失败时无论是 `pgx.ErrNoRows` 还是连接中断 / 类型不匹配，
    均返回 `ErrNotFound`，导致 DB 不可用时 HTTP 层返回 404 而非 503。
  - 修复：仅对 `errors.Is(err, pgx.ErrNoRows)` 返回 `ErrNotFound`，其余错误原样返回。
  - 文件：`services/api/internal/userplaylist/postgres/repository.go`

- [ ] **fix(desktop): tray Quit 改用 Flutter 生命周期退出**
  - 根因：`exit(0)` 绕过 Flutter `WidgetsBindingObserver`，SQLite WAL 未 checkpoint，
    正在进行的下载留下孤立文件，dispose() 链不执行。
  - 修复：将 `exit(0)` 替换为
    `WidgetsBinding.instance.handleRequestAppExit()` 或
    `SystemNavigator.pop(animated: false)`，确保 dispose/cleanup 链正常执行。
  - 文件：`services/mobile/lib/src/shared/desktop_integration.dart`

- [ ] **fix(offline): _restoreFromDb 改为 AsyncNotifier 或加错误处理**
  - 根因：`build()` 以 fire-and-forget 方式调用 `_restoreFromDb()`，异常静默丢失；
    notifier 在 future 完成前被销毁时 `state =` 抛 `StateError`（同样被丢弃）。
  - 修复方案 A（推荐）：将 `DownloadNotifier` 改为 `AsyncNotifier`，在 `build()` 中
    `await _restoreFromDb()`，将恢复失败暴露为 AsyncError。  
    方案 B：保持同步 `build()`，在 `_restoreFromDb` 内加 `try/catch` 并记录错误，
    用 `ref.notifier.mounted` 守卫 `state =` 赋值。
  - 文件：`services/mobile/lib/src/offline/download_notifier.dart`

- [ ] **fix(offline): 下载失败时清理磁盘上的部分文件**
  - 根因：`catch` 块仅更新状态为 `DownloadError`，不删除已写入 `localPath` 的部分文件；
    若在 step 4/5（DB insert 前）失败，完整文件也会永久孤立。
  - 修复：在 catch 块中将 `localPath` 提升到 try 作用域外，执行
    `await File(localPath).delete()` 并忽略 `FileSystemException`。
  - 文件：`services/mobile/lib/src/offline/download_notifier.dart`

- [ ] **fix(router): /library/my-playlists 父路由渲染实际列表页**
  - 根因：父路由 `/library/my-playlists` 的 `builder` 返回 `SizedBox.shrink()`，
    直接导航或子路由 pop 后显示空白屏。
  - 修复：创建 `UserPlaylistListScreen`（展示 `userPlaylistProvider` 列表，
    可复用 `FavoritesScreen` 结构），替换 `SizedBox.shrink()` 作为父路由 widget。
  - 文件：
    - `services/mobile/lib/src/shared/router.dart`
    - `services/mobile/lib/src/user_playlist/user_playlist_list_screen.dart`（新建）

### Medium — 用户体验 / 错误处理

- [ ] **fix(user_playlist): rename() 暴露错误并回滚乐观更新**
  - 根因：`catch (_) {}` 吞掉所有异常；乐观更新在服务端拒绝后不回滚，下次启动时出现静默回退。
  - 修复：移除 `catch (_)`，改用 `state = AsyncError(e, st)` 或在 UI 层 catch 并展示
    SnackBar；`rename()` 的乐观更新改为"先备份旧状态，失败时恢复"。
  - 文件：
    - `services/mobile/lib/src/user_playlist/user_playlist_notifier.dart`
    - `services/mobile/lib/src/user_playlist/user_playlist_detail_screen.dart`

- [ ] **fix(router): _DeepLinkTrackScreen 加 try/catch 防止无限 spinner**
  - 根因：`playTrack()` 抛出时 `context.go` 不执行，用户在深链页面无限等待。
  - 修复：在 `addPostFrameCallback` 内 wrap `try/catch`，失败时展示错误 SnackBar
    并调用 `context.go(AppRoutes.home)` 回退到主页。
  - 文件：`services/mobile/lib/src/shared/router.dart`

- [ ] **fix(offline): OfflineDb 双重 open 竞态**
  - 根因：`_db ??= await _open()` 在 Dart async 模型下非原子，两个并发 await 均可进入 `_open()`。
  - 修复：用 `Completer<Database>` 实现单次初始化：
    ```dart
    Completer<Database>? _completer;
    Future<Database> get db async {
      _completer ??= Completer()..complete(_open());
      return _completer!.future;
    }
    ```
  - 文件：`services/mobile/lib/src/offline/offline_db.dart`

- [ ] **fix(user_playlist_detail): removeTrack onPressed 加 try/catch**
  - 根因：网络错误从 `onPressed` 传播为未处理异常，列表不刷新，无用户反馈。
  - 修复：包裹 `try/catch`，失败时展示错误 SnackBar，成功时调用 `_loadTracks()`。
  - 文件：`services/mobile/lib/src/user_playlist/user_playlist_detail_screen.dart`

- [ ] **fix(user_playlist_detail): _name 改为从 provider 派生，消除双数据源**
  - 根因：`_name` 本地 state 与 `userPlaylistProvider` 并存，rename 失败后
    `setState(() => _name = newName)` 无条件执行导致 AppBar 标题错误。
  - 修复：删除 `_name` 字段，改为 `ref.watch(userPlaylistProvider).value
    ?.firstWhereOrNull((p) => p.id == widget.playlistId)?.name ?? ''` 派生。
  - 文件：`services/mobile/lib/src/user_playlist/user_playlist_detail_screen.dart`

- [ ] **commit: 提交 5 个未追踪的测试文件**
  - 根因：`artwork_url_provider_test.dart`、`history_notifier_test.dart`、
    `login_screen_test.dart`、`player_notifier_test.dart`、
    `track_favorite_notifier_test.dart` 均为 `??` 状态，CI 无法执行。
  - 修复：`git add services/mobile/test/*.dart`，在本 phase commit 中一并提交。
  - 验证：`flutter test` 本地全绿后推送，CI flutter.yml 应全部通过。
