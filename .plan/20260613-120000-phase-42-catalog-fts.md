# Phase 42 - Catalog Full-Text Search

## Requirement

Add PostgreSQL full-text search over catalog metadata (artists, albums, tracks) so that
administrators can find catalog entities by name without knowing exact identifiers.

## Tasks

- [ ] Add migration `006_catalog_fts` — tsvector columns and GIN indexes for artists, albums, tracks; UPDATE triggers to keep tsvectors current.
- [ ] Add `SearchCatalog` to `catalog.Repository` interface and `catalog.Service`.
- [ ] Add `CatalogSearchResult` and `SearchResultItem` types to `catalog/types.go`.
- [ ] Implement `SearchCatalog` on `catalogpg.Repository` using `plainto_tsquery` with ranked results.
- [ ] Implement `SearchCatalog` on `MemoryRepository` with case-insensitive substring fallback.
- [ ] Add `GET /api/v1/admin/catalog/search?q=` HTTP handler; map empty `q` → 400.
- [ ] Add 5+ service unit tests for `SearchCatalog` via `MemoryRepository`.
- [ ] Add 4+ HTTP-layer tests for the search endpoint.
- [ ] Add integration test for `SearchCatalog` in `repository_integration_test.go`.
- [ ] Update OpenAPI contract with search path and response schema.
- [ ] Bump VERSION to 0.42.0.

## Non-goals

- Ranked cross-entity scoring — equal ts_rank per entity kind is sufficient.
- Pagination of search results (phase-level addition later).
- Fuzzy / similarity matching (pg_trgm extension) for 0.x.
- Public (unauthenticated) catalog search endpoint.

## Follow-up candidates

- Phase 43: user-facing read-only catalog browse endpoints.
- Phase 44: batch import endpoint.
- Phase 45: pg_trgm fuzzy search and ranked cross-entity scoring.
