# Phase v3.6.0 — 代码审计缺陷修复

**版本**: v3.6.0  
**状态**: done  
**方向**: Flutter 客户端三项审计缺陷修复

## 背景

v3.5.0 代码审计（2026-06-29）发现三处真实缺陷：
1. `UserPlaylistDetailScreen` "Play All" 仅跳转播放器但未触发实际播放队列
2. `AppRoutes.deepAlbum` / `deepArtist` 是死代码——对应 GoRouter 路由未注册
3. `DownloadNotifier.startDownload` 将 `track.artistId`（UUID）存为 `artistName`

## 任务清单

- [x] **fix: UserPlaylistDetailScreen Play All 补 playQueue 调用**
  - `onPressed` 改为 `ref.read(playerProvider.notifier).playQueue(_trackIds!)` 后再 `context.go(AppRoutes.player)`
  - 文件：`lib/src/user_playlist/user_playlist_detail_screen.dart`
- [x] **fix: 补齐 deepAlbum / deepArtist 顶层路由**
  - 新增 `_DeepLinkAlbumScreen`（navigates to AlbumDetailScreen via shell push）
  - 新增 `_DeepLinkArtistScreen`（navigates to ArtistDetailScreen via shell push）
  - 在 GoRouter 顶层注册 `/albums/:id` 和 `/artists/:id` 路由，重定向到 shell 内对应页面
  - 文件：`lib/src/shared/router.dart`
- [x] **fix: OfflineTrack.artistName 存真实名称**
  - `startDownload` 中调用 `catalogRepositoryProvider.getArtist(track.artistId)` 获取真实名称，失败时回退 artistId
  - 同理 `albumTitle` 调用 `getAlbum(track.albumId)` 获取真实标题
  - 文件：`lib/src/offline/download_notifier.dart`
