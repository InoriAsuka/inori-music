# Phase v4.7.0 — 播放器音频真实性修复

**版本**: v4.7.0
**状态**: planned
**方向**: Flutter 客户端（音频引擎核心缺陷修复）

## 背景

2026-07-06 全量代码审查（逐行核对 v4.1.0–v4.2.0 声称完成的音频特性）发现多项特性为
**假实现或存在严重缺陷**，requirement.md 记录与实际代码不符：

1. **EQ 均衡器无音频效果**：requirement v4.1.1 声称"引入 `just_audio_equalizer` 插件"，但
   `pubspec.yaml` 从未加入该依赖（仅有 `just_audio: ^0.9.44` / `just_audio_background` /
   `audio_service`）；`audio_handler.dart:103` 的 `applyEqualizerBands` 经
   `(_player as dynamic).setBands(bands)` 调用一个 `AudioPlayer` 上**不存在的方法**，
   NoSuchMethodError 被空 catch 吞掉。Settings 页的 10 条频段滑块（`settings_screen.dart`
   `_EqSection`）是纯 UI，对音频输出零影响。
2. **ReplayGain 从未被应用**：`replay_gain_notifier.dart:11` 注释声称"实际增益应用发生在
   PlayerNotifier.playTrack"，但 `player_notifier.dart` 全文没有任何代码读取
   `replayGainEnabledProvider` 或 track 的 `replayGainDb` 字段。v4.5.0 服务端 ffmpeg 自动
   分析出的响度值在客户端无消费方。
3. **Shuffle 是空操作**：`player_notifier.dart:291` `setShuffle` 只翻转 `state.shuffle`
   标志位；`next()`/`previous()` 始终按顺序索引推进，从不参考该标志。
4. **Crossfade 不是交叉淡化**：`audio_handler.dart:137` `_runCrossfade` 在
   `currentIndexStream` 变化**之后**对同一个 player 做音量 1→0→1 的 V 形凹陷——切歌后新曲目
   先被压低再抬回，不是两轨重叠淡化；与 requirement v4.2.0 声称的"clip model 实现"不符。
   它还会覆写用户音量、暂停时可能停在半音量。
5. **Gapless 队列播放状态脱轨（最严重）**：`PlayerNotifier` 不监听
   `_audioPlayer.currentIndexStream`。`ConcatenatingAudioSource` 内部自动切到下一曲时，
   `state.currentIndex`/`state.mediaItem`/通知栏元数据全部停留在旧曲目，`_postHistory`
   不触发（`processingStateStream.completed` 只在**整个队列**播完时触发一次）；队列播完后
   `next()` 的 `% queue.length` 取模使队列无限循环，无视 `RepeatMode.off`。
6. **切歌性能**：`_playAtIndex` → `playTrack` → `_buildConcatQueue` 每次手动切歌都串行
   重解析全队列 URL（N 个网络请求）并整体重建音源，大队列下明显卡顿，也破坏了 gapless 缓冲。
7. **v4.1.1 声称的"presigned URL 提前 120s 刷新（currentIndexStream 监听）"在代码中不存在**
   （全库仅 `initCrossfade` 一处使用 `currentIndexStream`）。

注意：v4.6.0 的「EQ 自定义预设保存与命名」只涉及设置数据持久化，与本阶段无冲突；
但只有本阶段完成 EQ 真实接线后，那些预设才真正可听。建议 v4.6.0 → v4.7.0 顺序执行
（两者都改 `player_notifier.dart`，避免并行冲突）。

## 目标

- 队列 gapless 自动前进时 UI/通知栏/历史上报与实际播放曲目始终一致
- RepeatMode.off 播完停止、all 循环、one 单曲循环，语义正确
- ReplayGain 开关打开时实际调整播放增益
- Shuffle 开关实际改变播放顺序
- EQ 在 Android 上真实作用于音频输出；不支持的平台明确标注禁用而非静默无效
- Crossfade 设置项行为与其名称/文档一致
- 手动切歌不再全队列重建，即点即切

## 任务清单

### Gapless 状态同步（`lib/src/player/player_notifier.dart`）

- [ ] `_setupPlayerListeners` 新增 `_audioPlayer.currentIndexStream` 订阅：concat 源索引变化时
      更新 `state.currentIndex`、解析并更新 `state.mediaItem`、推送 `audioHandler.mediaItem`
- [ ] 历史上报改为在 index 前进时对**刚完成的曲目**调用 `_postHistory`（携带其 trackId，
      不再依赖 `processingStateStream.completed`）；completed 仍处理最后一曲的上报
- [ ] repeat 语义修复：`RepeatMode.one` → `_audioPlayer.setLoopMode(LoopMode.one)`；
      `RepeatMode.all` → `LoopMode.all`；`RepeatMode.off` → `LoopMode.off` 且队列播完自然停止；
      删除 `next()` 中无条件 `% queue.length` 的回绕
- [ ] 手动 `next()`/`previous()` 改用 `_audioPlayer.seekToNext()`/`seekToPrevious()`
      （concat 源已加载时），不再经 `_playAtIndex → playTrack` 重建音源
- [ ] `enqueue`/`enqueueNext`/`removeFromQueue`/`reorderQueue` 改为对
      `ConcatenatingAudioSource` 做 `add`/`insert`/`removeAt`/`move` 增量操作
      （`audio_handler.dart` 暴露对应方法），移除 `_refreshConcatSource` 全量重建
- [ ] `resolvePlaybackUrl` 的相对路径 fallback（`'/api/v1/...'` 字符串，`player_notifier.dart:98`）
      修正为拼接 base URL，或删除该不可达分支

### ReplayGain 实际应用（`lib/src/player/player_notifier.dart`）

- [ ] 曲目开始播放/索引变化时读取 `replayGainEnabledProvider` 与 `_trackCache[id]?.replayGainDb`，
      计算 `gain = pow(10, db / 20).clamp(0.1, 2.0)`，与用户音量相乘后 `setVolume`
- [ ] 用户音量与增益分离：`state.volume` 保存用户意图值，实际 `setVolume(user * gain)`；
      开关切换即时生效
- [ ] `replayGainDb` 为 null 时增益取 1.0（无分析数据不变化）

### Shuffle 实现（`lib/src/player/player_notifier.dart`）

- [ ] `setShuffle(true)`：`_audioPlayer.setShuffleModeEnabled(true)`（concat 源原生支持），

- [ ] QueueSheet/队列 UI 按 `shuffleIndicesStream` 呈现实际播放顺序
