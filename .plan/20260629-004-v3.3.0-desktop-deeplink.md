# Phase v3.3.0 — 桌面平台增强 + 深度链接

**版本**: v3.3.0  
**状态**: planned  
**方向**: 桌面平台功能 + 移动端深度链接

## 背景

v3.0.0 的桌面端（macOS/Windows/Linux）仅有自适应布局和键盘快捷键，缺少系统级集成（托盘、全局快捷键）。移动端也无法从外部链接打开指定曲目。

## 任务清单

### 桌面系统集成

- [ ] 引入 `package:tray_manager ^0.2.3`
  - macOS/Windows/Linux：`TrayManager.instance.init('assets/images/tray_icon.png')`
  - 托盘菜单：Play/Pause / Next / Open Window / Quit
  - `TrayListener.onTrayMenuItemClick` → 调 `playerProvider.notifier`
- [ ] 引入 `package:hotkey_manager ^0.2.0`（仅 desktop）
  - 注册全局快捷键（`HotKeyManager.instance.register`）：
    - `Space` → togglePlayPause
    - `ArrowRight` → next
    - `ArrowLeft` → previous
    - `MediaPlay/Pause`（多媒体键）→ togglePlayPause
  - `AppLifecycleListener.onDetach` 时 unregister
- [ ] `pubspec.yaml` platform-conditional 依赖（`flutter.platforms: [macos, windows, linux]`）
- [ ] CI：macOS runner 验证 `flutter build macos --release`

### 移动端深度链接

- [ ] Android：`android/app/src/main/AndroidManifest.xml`
  - `<intent-filter>` 添加 `inori://` scheme + `https://` App Link
  - 路径映射：`inori://tracks/{id}` / `inori://albums/{id}` / `inori://artists/{id}`
- [ ] iOS：`ios/Runner/Info.plist`
  - `CFBundleURLSchemes: [inori]`
  - `NSUserActivityTypes` + Associated Domains（需服务端 `.well-known/apple-app-site-association`）
- [ ] go_router 处理 deep link
  - `GoRouter.initialLocation` 来自 `GoRouterState`
  - 路径 `inori://tracks/{id}` → 先 push `/tracks`，再播放该 track
- [ ] 测试：Android `adb shell am start -a android.intent.action.VIEW -d "inori://tracks/xxx"`
