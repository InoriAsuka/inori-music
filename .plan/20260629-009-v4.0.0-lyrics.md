# Phase v4.0.0 — 歌词支持（LRC / SRT 同步滚动）

**版本**: v4.0.0  
**状态**: planned  
**方向**: 全栈（Go 服务端新 asset kind + Flutter 歌词渲染层）

## 背景

v3.6.1 的 `FullPlayerScreen` 封面区下方有歌词占位文字，但无实际数据源。
服务端无 `lyrics` asset kind，无上传/获取端点；Flutter 端无解析器和同步滚动实现。
v4.0.0 将歌词从零到一打通：管理员上传 LRC/SRT → 服务端存储 → Flutter 实时同步高亮。

## 目标

- 服务端可存储、提供、删除 LRC/SRT 格式歌词文件
- Flutter `FullPlayerScreen` 展示同步滚动高亮歌词，随播放位置实时定位
- 无歌词曲目展示占位，不影响现有播放体验

## 任务清单

### 服务端（Go `services/api/`）

- [ ] **新增 asset kind `lyrics`**
  - 在 `internal/storage` 的 AssetKind 枚举中追加 `AssetKindLyrics = "lyrics"`
  - 文件：`services/api/internal/storage/asset_kind.go`（或同等位置）

- [ ] **Catalog Track 关联歌词**
  - `catalog.Track` 新增 `LyricsMediaObjectID string`（omitempty）
  - `tracks` 表 migration `012_track_lyrics`：`ALTER TABLE tracks ADD COLUMN IF NOT EXISTS lyrics_media_object_id TEXT`
  - `catalogpg.Repository`：Save/Get/List/Scan 同步新增列
  - 文件：
    - `services/api/internal/catalog/track.go`
    - `services/api/internal/catalog/postgres/repository.go`
    - `services/api/migrations/012_track_lyrics.sql`

- [ ] **`POST /api/v1/catalog/tracks/{id}/lyrics`**（requireAdminAuth）
  - 接受 `multipart/form-data`（`file` 字段，Content-Type `text/plain` 或 `application/octet-stream`）
  - 验证格式：LRC（首行含 `[ti:]` 或 `[00:00.00]`）或 SRT（首行为数字序号）；非法格式返回 400
  - 调 `storage.Service.PutObject(kind=lyrics, ...)` 存储，返回 media object ID
  - 更新 `track.LyricsMediaObjectID`，调 `catalogService.UpdateTrack`
  - 成功返回 `201 { "mediaObjectId": "..." }`

- [ ] **`GET /api/v1/catalog/tracks/{id}/lyrics`**（requireViewerAuth）
  - 查 `track.LyricsMediaObjectID`，无值返回 404
  - 调 `storage.Service.GetObject(...)` 读取文件内容（UTF-8 文本，≤ 512KB）
  - 解析格式（lrc / srt）
  - 响应：`200 LyricsResponse{ format: "lrc"|"srt", content: "<原始文本>" }`
  - 存储错误 → 500

- [ ] **`DELETE /api/v1/catalog/tracks/{id}/lyrics`**（requireAdminAuth）
  - 清除 `track.LyricsMediaObjectID`（置 null）
  - 调 `storage.Service.DeleteObject` 删除媒体对象
  - 成功返回 204

- [ ] **OpenAPI contract 更新**
  - 新增 `LyricsResponse` schema（`format: enum[lrc, srt]`, `content: string`, `mediaObjectId: string`）
  - 新增三条路径（POST/GET/DELETE `/api/v1/catalog/tracks/{id}/lyrics`）
  - `CatalogTrack` schema 新增 `lyricsMediaObjectId`（string, nullable）
  - 版本升至 `4.0.0`
  - 文件：`packages/api-contract/openapi/storage-admin.v1.json`

- [ ] **单元测试**
  - `getLyrics` handler：成功返回 content / 无歌词 404 / 存储错误 500
  - `uploadLyrics` handler：成功 201 / 格式校验失败 400
  - 文件：`services/api/internal/httpapi/handler_test.go`（或对应 lyrics 测试文件）

### Flutter 客户端（`services/mobile/`）

- [ ] **LRC/SRT 解析器**
  - `lib/src/lyrics/lyric_line.dart`：`class LyricLine { final Duration timestamp; final String text; }`
  - `lib/src/lyrics/lrc_parser.dart`：解析 `[MM:SS.xx]` 时间戳行，过滤元数据行（`[ti:]` 等），返回 `List<LyricLine>` 按 timestamp 排序
  - `lib/src/lyrics/srt_parser.dart`：解析 SRT 序号块（序号 / 时间轴 `HH:MM:SS,mmm --> HH:MM:SS,mmm` / 文本），返回 `List<LyricLine>`（起始时间戳）

- [ ] **`lyricsProvider(trackId)`**
  - `AutoDisposeFamilyAsyncNotifier<List<LyricLine>?, String>`
  - 调用 `GET /api/v1/catalog/tracks/{id}/lyrics`
  - 根据 `format` 字段选择对应解析器
  - 404 → 返回 `null`（无歌词）；其他错误抛出
  - 文件：`lib/src/lyrics/lyrics_provider.dart`

- [ ] **`FullPlayerScreen` 歌词层**
  - 封面区与歌词区通过 `PageView`（水平滑动）切换，底部 Page Indicator
  - 歌词页：`ListView.builder` + `ScrollController`，当前行 = `lines.lastWhere(l => l.timestamp <= position)`
  - 当前行：字号加大（16 → 18sp）、primary color 高亮、`ScrollController.animateTo` 滚至居中
  - 非当前行：`onSurfaceVariant` 色，opacity 0.6
  - 监听 `playerProvider.select((s) => s.position)`，防抖 200ms（避免每帧重建）
  - 无歌词（`lyricsProvider == null`）：展示 `"暂无歌词"` 占位文字（居中，onSurfaceVariant 色）
  - 文件：`lib/src/player/full_player_screen.dart`

- [ ] **flutter analyze 0 errors**
