# Phase v4.8.0 — 测试与结构还债（v4 封版收官）

**版本**: v4.8.0
**状态**: planned
**方向**: 全栈（Go 结构重构 + 三端测试基建 + 仓库卫生）

## 背景

2026-07-06 全量审查量化了当前的债务分布：

- **测试严重失衡**：Go API 有 29 个测试文件 / 773 个测试（CI 含 `-race`）✅；
  但 Flutter 465 个源文件仅 12 个测试文件，Web（~14K 行）只有 1 个 Playwright smoke spec
  （`services/web/e2e/smoke.spec.ts`）且无单测，Admin（~13K 行）**零测试**。
  缺陷密度最高的移动端播放器恰好是测试最薄的地方（v4.7.0 的五个假实现任何一条
  播放集成测试都能抓住）。
- **上帝文件**：`services/api/internal/httpapi/handler.go` 5218 行、
  `handler_test.go` 9255 行，storage/media/auth/catalog/history/favorites/userplaylist/search
  八个域的 HTTP 层挤在一个文件里。
- **文档漂移**：`README.md` 停留在 v3.0.1 基线，技术栈描述错误（写 React+Vite，实际
  Next.js 15；路径写 `packages/web/`，实际 `services/web/`），Phase 记录断在 313。
- **版本号脱钩**：`services/web/package.json` 2.6.0、`services/admin/package.json` 2.3.0，
  与 VERSION 4.x 无关联。
- **仓库卫生**：`.codegraph/daemon.pid` 在 72d63ee 加入 .gitignore 后仍被 git 跟踪
  （需 `git rm --cached`），每次会话产生噪音 diff。
- CI 的 web/admin job 只跑 `type-check`，`biome lint` 脚本存在但未进 CI。

本阶段完成后 v4 线封版：v4.x 不再规划新 Phase，后续仅接受 bug 修复 patch（v4.8.x）。

## 目标

- handler.go 按域拆分为多个文件，单文件不超过 ~1000 行，行为零变更（773 测试护航）
- Flutter/Web/Admin 都具备可持续追加的测试基建，主流程有回归保护
- README 与实际代码一致，新人按 README 能跑起全栈
- 版本号、gitignore、CI lint 等卫生项清零
