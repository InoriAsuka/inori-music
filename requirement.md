# inori-music 中文需求文档

## 当前版本

`0.22.0`

## 总体目标

构建全平台音乐播放系统，支持 Web、Android、iOS、PC 等客户端，同时兼容 B/S 与 C/S 架构。服务端负责媒体存储配置、元数据登记、健康探测、完整性校验与管理接口；大型媒体字节保存在外部存储后端中。

## 技术栈要求

- 客户端优先 Flutter。
- 服务端优先 Go 模块化单体。
- 服务端元数据数据库以 PostgreSQL 为 0.x 首选。
- 客户端本地数据库采用 SQLite。
- 0.x 搜索优先 PostgreSQL 全文检索，后续保留外部搜索引擎扩展能力。
- 媒体存储支持本地文件系统、NFS、SMB、S3 兼容对象存储和分布式存储适配。

## 存储要求

- 不将大型音频、图片或派生文件写入关系型数据库。
- 数据库仅保存对象 ID、后端 ID、对象键、哈希、生命周期、资产类型、校验状态和引用关系。
- 所有探测与校验必须采用服务端拥有的临时对象或只读方式，避免破坏用户媒体数据。

## 文档要求

- Markdown 文档统一使用中文维护。
- README 拆分为 `README.md` 入口和 `README_zh.md` 中文完整说明。
- 每个阶段必须在 `.plan/` 下记录计划、任务清单、非目标和后续候选项。
