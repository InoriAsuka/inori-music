# 存储后端架构

## 目标

服务端统一管理媒体存储配置，屏蔽本地文件系统、NFS、SMB、S3 兼容对象存储和分布式存储之间的差异。

## 后端类型

- `local`：开发、单节点自托管或已挂载 NAS 路径。
- `nfs`：局域网或家庭实验室中的 NFS 挂载目录。
- `smb`：Windows 共享、NAS 设备和混合平台网络中的 SMB/CIFS 挂载目录。
- `s3`：云厂商或自托管对象存储提供的 S3 兼容接口。
- `distributed`：Ceph、Garage、SeaweedFS 等分布式存储，可通过 S3 兼容接口、挂载文件系统或未来专用适配器接入。

## 能力模型

后端通过能力集合描述是否支持服务端范围读取、预签名 URL、分片上传、原生生命周期策略、跨节点访问、挂载验证和凭据验证。能力由静态配置推导，并可通过探测结果持续修正健康状态。

## 安全原则

探测只能创建服务端拥有的临时文件或对象，并在完成后清理。应用不负责挂载或卸载 NFS/SMB；凭据通过环境变量引用解析，不能将真实密钥写入仓储文件。
